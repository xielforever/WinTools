# WinTools 发布说明

最后更新：2026-03-12

## 1. 版本规范

- 发布标签必须是 `vMAJOR.MINOR.PATCH`，例如：`v0.1.0`
- 不符合格式的版本（如 `1.0.0`、`v1.0`、`v1.0.0-rc.1`）会在工作流“版本校验”步骤直接失败
- Git Tag 是唯一发布版本来源

## 2. 本地构建

在仓库根目录执行：

```powershell
./scripts/build.ps1 -Version v0.1.0
```

脚本行为：

- 清理旧构建目录（`dist/`、`build/`）
- 安装构建依赖（PyInstaller）
- 构建 OneDir 包并压缩为 ZIP
- 构建 OneFile EXE

输出产物：

- `dist/WinTools-v0.1.0-windows-onedir.zip`
- `dist/WinTools-v0.1.0-windows-onefile.exe`

## 3. GitHub Release 自动发布

仓库已配置工作流：`.github/workflows/release.yml`

触发方式：

- 推送标签触发：`git push origin v0.1.0`
- 手动触发：GitHub -> Actions -> `Release` -> `Run workflow`，输入 `version_tag`

发布流程：

1. 校验版本标签格式
2. 安装 Python 3.13 与构建依赖
3. 执行 `scripts/build.ps1` 产出双产物
4. 上传 Actions Artifacts
5. 创建/更新 GitHub Release，并附加产物
6. Release 文案采用“自动生成 notes + 固定安装说明模板”

## 4. 常见失败排查

### 4.1 版本校验失败

- 现象：提示 `Invalid version tag`
- 原因：标签格式不符合 `vMAJOR.MINOR.PATCH`
- 处理：改用类似 `v0.1.0` 的标签

### 4.2 打包失败（PyInstaller）

- 现象：构建步骤报错
- 处理：
  - 确认使用 Python 3.13
  - 重新安装构建依赖：`python -m pip install -e ".[build]"`
  - 本地先执行 `./scripts/build.ps1 -Version v0.1.0` 复现并定位

### 4.3 Release 创建失败（权限）

- 现象：发布步骤无权限写入 Release
- 处理：确认仓库 Actions 权限允许 `contents: write`

## 5. 当前边界

- 当前仅面向 Windows 发布
- 首版不包含 Installer（Inno Setup/MSIX）
- 当前不包含代码签名与自动升级机制
